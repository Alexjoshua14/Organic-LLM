# Security Policy

## Supported versions

Security fixes are applied to the default branch (`main`) and release tags when they exist. Older branches may not receive patches.

## Reporting a vulnerability

If you discover a security issue, please **do not** open a public GitHub issue with exploit details.

Email **coalescencelabs@gmail.com** with:

- A description of the issue and impact
- Steps to reproduce (proof-of-concept if available)
- Affected paths, versions, or commit SHA when known

We aim to acknowledge reports within a few business days. Please allow reasonable time for a fix before public disclosure; we will coordinate disclosure with you when possible.

## What not to commit

Never commit secrets or production credentials to this repository, including:

| Category | Examples |
|----------|----------|
| Auth | `CLERK_SECRET_KEY`, `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` (treat publishable keys as project-specific) |
| Database | `SUPABASE_SERVICE_ROLE_KEY`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, connection strings with passwords |
| Encryption | `ORGANIC_LLM_ROOT_SECRET`, `ORGANIC_LLM_KEY_REGISTRY_JSON`, `MEMORY_ENCRYPTION_*` |
| APIs | `OPENAI_API_KEY`, `EXA_API_KEY`, `ELEVENLABS_API_KEY`, `MEMORY_API_SECRET` |
| Infra | `UPSTASH_REDIS_REST_TOKEN`, `CRON_SECRET`, `OLLAMA_API_KEY` |
| Local | `.env.local`, `.env*.local`, `.local-profile-overrides.ts` |

Use `.env.local` (gitignored) for local development. See `.env.example` for variable names only—no real values.

## Local-only paths

These are generated on your machine and must stay out of git:

- `supabase/.temp/` — Supabase CLI link state (regenerated by `supabase link`)
- `test-results/` — Playwright output
- `.chats/`, `memory.db` — local data (see `.gitignore`)

## Scope notes

- **Demo routes** (`/showcase`, `/blog`, etc.) are intentionally public; report issues that leak auth-bound data or bypass access controls on protected routes (`/chat`, `/api/*`, etc.).
- Dependency vulnerabilities: we use `bun` lockfile and GitHub Dependabot/alerts when enabled; report supply-chain issues with the affected package and version.

Thank you for helping keep Organic LLM and its users safe.
